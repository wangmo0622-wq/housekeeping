# 小程序接口文档

## 基础信息

- Base URL: `/api`
- 认证方式: JWT Bearer Token
- Content-Type: `application/json`

### 1.3 分类树

获取分类树结构。**仅包含后台状态为「启用」的分类**。树形语义：**顶层元素为一级分类**，其 **`children` 内为二级分类**（叶子节点的 `children` 固定为空数组 `[]`）。后台用整型 **`parent_id`** 存储父子关系（**`0` 表示一级**；二级为父级 `id`），业务上**禁止三级**；**本接口响应中不返回 `parent_id`**，小程序只使用 **`id` + `children`** 即可，**无需因存储方式调整而改前端代码**。**后台「分类管理」创建/编辑时已限制为最多两级**（一级根 + 二级子类）。同级节点顺序与后台一致：先按 `sort_order` 升序，相同则按 `id` 升序。

每个节点除 `id`、`name`、`children` 外，还包含 **`sort_order`**（整数，与后台排序一致，便于客户端自定义展示顺序）。

**接口地址:** `GET /api/public/categories/tree`

**请求参数:** 无

**响应示例:**
```json
{
  "tree": [
    {
      "id": 1,
      "name": "家政服务",
      "sort_order": 0,
      "children": [
        {
          "id": 2,
          "name": "保洁",
          "sort_order": 0,
          "children": []
        }
      ]
    }
  ]
}
```

**字段说明:**
| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | int | 分类ID |
| name | string | 分类名称 |
| sort_order | int | 排序值，越小越靠前（与后台一致） |
| children | array | 子分类数组（递归结构；叶子节点为 `[]`） |

**说明:** 响应体顶层键名为 `tree`，与后台「停用」分类不会出现在树中；若父级被停用，其下已启用的子级也不会出现在本接口结果中（小程序侧无法从该树访问到这些节点）。

**缓存（重要）：**
- **服务端**：默认把树 JSON 缓存在 Redis 等后端里约 **300 秒**（环境变量 **`PUBLIC_CATEGORY_TREE_CACHE_SECONDS`**）。**后台保存/删除分类** 时会**主动删掉**这份缓存，下一次请求会重新查库。
- **HTTP 与开发者工具**：当该值 **> 0** 时，响应里会有 **`Cache-Control: max-age=…`**，微信开发者工具、部分真机 HTTP 栈可能对 **GET** 做**本地缓存**，即使服务端已更新，仍可能短暂看到**旧树**。联调分类请设 **`PUBLIC_CATEGORY_TREE_CACHE_SECONDS=0`**（服务端不落缓存，且响应为 **no-store / no-cache**），或在工具里勾选「不校验合法域名…」旁相关缓存清理、重新编译。
- 设为 **0** 会加重数据库读取，仅建议开发/排障或分类变更极频繁阶段使用。

### 1.6 服务列表

获取服务列表，支持筛选和分页

**接口地址:** `GET /api/public/listings`

**请求参数:**
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| category_id | int | 否 | 分类ID |
| service_type_id | int | 否 | 服务类型ID |
| min_price | float | 否 | 最低价格 |
| max_price | float | 否 | 最高价格 |
| ordering | string | 否 | 排序方式，默认 `-published_at` |
| technician_id | int | 否 | 技师ID，筛选指定技师的服务 |
| search | string | 否 | 关键词，对**服务标题**、**服务描述**做模糊匹配（二者任一包含即命中）；可与分类、价格等条件**同时生效**（AND） |
| q | string | 否 | 与 `search` **含义相同**，任选其一传入即可；若同时传 `search` 与 `q`，二者内容一致时效果不变，建议只传一个 |
| page | int | 否 | 页码，默认 1 |
| page_size | int | 否 | 每页数量，默认 10 |

**ordering 可选值:**
- `-published_at`: 发布时间倒序（默认）
- `-created_at`: 创建时间倒序
- `published_at`: 发布时间正序
- `created_at`: 创建时间正序

**响应示例:**
```json
{
  "count": 100,
  "next": "http://example.com/api/public/listings?page=2",
  "previous": null,
  "results": [
    {
      "id": 1,
      "title": "专业保洁服务",
      "description": "提供专业家政保洁服务",
      "cover_url": "https://example.com/cover1.jpg",
      "cover_urls": [
        "https://example.com/cover1.jpg",
        "https://example.com/cover2.jpg"
      ],
      "listing_price": "199.00",
      "listing_price_unit": "次",
      "service_price": "199.00 次",
      "category_id": 1,
      "category_name": "家政服务",
      "category_path": "家政服务 > 保洁",
      "primary_category_id": 1,
      "primary_category_name": "家政服务",
      "secondary_category_id": 2,
      "secondary_category_name": "保洁",
      "filter_category_id": 2,
      "secondary_categories": [
        { "id": 2, "name": "保洁" }
      ],
      "technician_id": 1,
      "technician_name": "张师傅",
      "technician_avatar": "https://example.com/avatar.jpg",
      "services": [
        {
          "id": 1,

          "base_price": "50.00",

      "created_at": "2024-01-01T10:00:00Z"
    }
  ]
}
```

**列表项补充字段说明:**

| 字段名 | 类型 | 说明 |
|--------|------|------|
| cover_url | string | 首张封面（与 `cover_urls[0]` 一致，兼容旧版） |
| cover_urls | string[] | 封面图 URL 列表，**最多 6 张** |
| category_id | int | **一级**分类ID（与 `primary_category_id` 相同，历史兼容字段） |
| category_name | string | 一级分类名称（稳定口径） |
| category_path | string | 展示路径，格式 `一级 > 二级` |
| primary_category_id | int | 一级分类ID（与 `category_id` 一致） |
| primary_category_name | string | 一级分类名称（与 `category_name` 一致） |
| filter_category_id | int / null | **推荐用于列表筛选**：传给 `GET /api/public/listings?category_id=` 时优先用本字段；有二级时为**二级 id**，否则为一级 id。若误用 `category_id` 只传一级，会筛出该一级下**全部二级**的服务，范围更大 |
| secondary_category_id | int / null | 兼容字段：第一个二级分类ID |
| secondary_category_name | string / null | 兼容字段：第一个二级分类名称 |
| secondary_categories | array | 二级分类数组（推荐使用） |
| listing_price | string / null | 技师填写的服务标价（未填则为 `null`） |
| listing_price_unit | string | 标价单位，默认 `次` |
| service_price | string / null | 展示用价格文案：优先一口价（`listing_price` + 单位）；未填时由关联服务类型的 `base_price` 等拼出区间或单价 |

### 2.2 手机号注册

使用手机号和密码注册新用户

**接口地址:** `POST /api/accounts/phone-register`

**请求参数:**
```json
{
  "phone": "13800138000",
  "password": "123456",
  "nickname": "用户昵称"
}
```

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| phone | string | 是 | 手机号（11位） |
| password | string | 是 | 密码（至少6位） |
| nickname | string | 否 | 用户昵称 |

**响应示例:**
```json
{
  "access": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refresh": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "user": {
    "id": 1,
    "username": "用户_68923",
    "nickname": "用户昵称",
    "phone": "13800138000",
    "is_technician": false,
    "technician_status": null,
    "technician_id": null
  }
}
```

**说明:**
- 用户名自动生成，格式：用户_ + 手机尾4位 + 1位随机数字（例：用户_68923）
- 手机号存储在技师资料中，用于登录和身份验证

**错误响应:**
```json
{
  "detail": "该手机号已注册"
}
```

| 错误信息 | 说明 |
|---------|------|
| 手机号不能为空 | 未提供手机号 |
| 密码不能为空 | 未提供密码 |
| 密码长度不能少于6位 | 密码太短 |
| 手机号格式不正确 | 手机号格式错误 |
| 该手机号已注册 | 手机号已被使用 |
