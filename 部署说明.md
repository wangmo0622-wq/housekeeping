# 虔诚家政 Docker 部署说明

## 域名配置
- **域名**: c.ningduhr.com
- **部署方式**: Docker + Docker Compose

## 服务器要求
- 已安装 Docker 和 Docker Compose
- 域名已解析到服务器 IP

## 部署步骤

### 1. 上传代码到服务器

```bash
# 在服务器上执行
cd /www/wwwroot
git clone <你的仓库地址> jiazheng
cd jiazheng
```

### 2. 配置环境变量

```bash
cp .env.example .env
# 编辑 .env 文件，确保以下配置正确
```

**.env 文件内容:**
```env
DEBUG=False
SECRET_KEY=你的随机安全密钥（至少50位随机字符）
ALLOWED_HOSTS=c.ningduhr.com,localhost,127.0.0.1
TIME_ZONE=Asia/Shanghai
LANGUAGE_CODE=zh-hans
```

### 3. 执行部署

```bash
./deploy.sh
```

### 4. 配置域名解析

在域名管理后台添加 A 记录：
- **主机记录**: `c`
- **记录类型**: `A`
- **记录值**: `你的服务器IP地址`

### 5. 配置 HTTPS（SSL证书）

#### 方式一：使用 Certbot 自动申请（推荐）

```bash
# 进入容器
docker-compose exec nginx sh

# 安装 certbot
apk add --no-cache certbot

# 申请证书
certbot --nginx -d c.ningduhr.com

# 退出容器
exit

# 重启 nginx
docker-compose restart nginx
```

#### 方式二：手动配置证书

1. 将 SSL 证书文件放入 `nginx/ssl/` 目录：
   - `nginx/ssl/c.ningduhr.com.crt` (证书)
   - `nginx/ssl/c.ningduhr.com.key` (私钥)

2. 更新 `nginx/nginx.conf` 添加 HTTPS 配置

## 访问地址

- **网站**: http://c.ningduhr.com

## 常用命令

```bash
# 查看日志
docker-compose logs -f

# 重启服务
docker-compose restart

# 停止服务
docker-compose down

# 进入 Django 容器
docker-compose exec backend bash

# 数据库迁移
docker-compose exec backend python manage.py migrate

# 创建超级用户
docker-compose exec backend python manage.py createsuperuser

# 收集静态文件
docker-compose exec backend python manage.py collectstatic --noinput
```

## 文件说明

| 文件 | 说明 |
|------|------|
| docker-compose.yml | Docker Compose 配置文件 |
| Dockerfile | Django 应用镜像构建文件 |
| nginx/nginx.conf | Nginx 反向代理配置 |
| .env | 环境变量配置（需手动创建） |
| deploy.sh | 一键部署脚本 |

## 注意事项

1. **SECRET_KEY**: 生产环境必须修改，使用随机生成的长字符串
2. **防火墙**: 确保服务器开放 80 和 443 端口
3. **备份**: 定期备份 PostgreSQL 数据（如 `docker compose exec db pg_dump -U …`）与 `media/` 目录
